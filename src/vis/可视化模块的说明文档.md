# vis 模块说明

## 概览
    主要入口：`src/vis/main.js`  
    子模块在 `src/vis/lib/`）
介绍：基于 Cesium 与 ECharts 展示时序网络节点（卫星/地面站/无人机）三维轨迹、实体模型与二维拓扑分析视图，并解析本地 CSV 数据（拓扑、轨迹、卫星）。

## 使用说明
- 启动：确保项目依赖安装完毕（`npm install`），使用开发服务器（例如 `npm run dev` 或 `vite`）在浏览器中打开页面。
- 数据文件：默认从根路径请求 `/mock_topology.csv`, `/mock_trace.csv`, `/mock_sat.csv`,读取三个csv文件来构建整个页面。
- 启用 3D 连线：取消 `src/vis/lib/initSystem.js` 中对 `create3DLinks` 的注释并确保 `links3d.js` 被正确导入与调用；但是可能影响性能。

## 注意事项
- 性能：三维全连线有 O(n^2) 的多边绘制开销，二维拓扑通过 `lastEdgesString` 做了去重优化。
- 时间同步：`shared.startUtc` 为固定起始时间，可以通过修改宏来改起点时间。
- 模型资源：使用 `models/` 目录下的本地 glb 模型（satellite.glb / gs.glb / uav.glb）

## 各个子模块功能

- **`src/vis/main.js`**
  - 功能：应用入口，初始化 Cesium Viewer、绑定全局交互函数（视图切换、选择目标、切换分析面板），并调用 `initSystem` 完成数据载入与实体创建。
  - 暴露/使用：通过 `initSystem(viewer)` 启动；全局函数 `toggleOption(type)` 控制分析面板显示，`selectTarget(id)` 切换追踪目标。

- **`src/vis/lib/initSystem.js`**
  - 功能：系统初始化核心模块：
    - 加载 Cesium 世界地形并启用地形深度测试；
    - 从 `/mock_topology.csv`, `/mock_trace.csv`, `/mock_sat.csv` 读取并解析数据（使用 PapaParse）；
    - 构建 `shared.topologyEvents`、`shared.nodeInfoList`；
    - 根据轨迹/卫星/地面站数据创建 Cesium 实体并添加到 `viewer.entities`；
  - 使用：由 `main.js` 调用 `initSystem(viewer)`。
  - 3D 连线功能（`create3DLinks`）默认被注释，因为web端性能有限，三维的连线开销比较大。

- **`src/vis/lib/links3d.js`**
  - 功能：构建并维护三维连线（PolylineCollection），在每个渲染周期更新连线可见性、位置和颜色，颜色/可见性基于链路状态（通过 `getLinkStatus` 获得）。
  - 使用：由 `initSystem` 可选启用（当前为注释），并会在每帧的 `scene.preRender` 中根据时间刷新状态；当 `shared.state.showAnalytics` 为真时会触发二维拓扑刷新。
  - 暂时没有启用理由同上

- **`src/vis/lib/state.js`**
  - 功能：共享状态容器，导出 `shared` 对象，包含：
    - `startUtc`：起始时间（JulianDate），
    - `state`：UI 状态（showTopology、showAnalytics、currentTarget、viewMode），
    - `entityMap`：id -> Cesium Entity 映射，
    - `topologyEvents`：按时间序列的链路事件，
    - `nodeInfoList`：节点基本信息列表，
    - `chart2D`：ECharts 实例引用，
    - `lastEdgesString`：用于避免重复渲染的边哈希。
  - 各模块通过 `import { shared } from './state.js'` 访问/更新全局状态。

- **`src/vis/lib/topo2d.js`**
  - 功能：基于 ECharts 绘制二维网络拓扑视图（力导向图），并根据当前时间与 `shared.topologyEvents` 显示链路状态。
  - 主要接口：`update2DTopology(viewer)`，在 `viewer` 时钟推进或分析面板显示时被调用以重建/更新图表数据。
  - 使用细节：首次调用会初始化 `shared.chart2D`；后续调用根据 `shared.nodeInfoList` 和 `getLinkStatus` 生成边集合并更新图表，仅在边集合发生变化时重绘以节省性能。

- **`src/vis/lib/utils.js`**
  - 功能：工具函数集合，目前包含 `getLinkStatus(idA, idB, currentTimeMs)`：
    - 根据 `shared.topologyEvents` 中与 idA/idB 对应的事件，查找在当前时间点之前最近的事件并返回其 `status`（如包含 'up' 等标识）。
  - 使用：被 `links3d.js` 与 `topo2d.js` 调用以判断链路是否可用及其状态字符串。



## 一些关键的函数

通过一些关键的函数来调整整个可视化的一些布局等。

- **`initSystem(viewer)`（`src/vis/lib/initSystem.js`）**
  - 概要：系统启动入口，完成地形加载、CSV 解析、实体创建、时钟设置与节点下拉刷新。
  - 关键代码与解释：

```js
// 加载世界地形并开启深度测试
viewer.terrainProvider = await Cesium.createWorldTerrainAsync();
viewer.scene.globe.depthTestAgainstTerrain = true;
```
  - 说明：使用 Cesium 世界地形可以恢复真实地形起伏，`depthTestAgainstTerrain` 确保实体与地形深度对齐。

```js
Papa.parse(topoRes, { header: true, dynamicTyping: true, skipEmptyLines: true }).data.forEach(row => { ... });
```
  - 说明：使用 PapaParse 解析 CSV，构建 `shared.topologyEvents`。事件包含 `time`、`key`与 `status` 等字段，后续链路判断以此为依据。

```js
const posProp = new Cesium.SampledPositionProperty();
posProp.addSample(t, rawPos);
viewer.entities.add({ id, position: posProp, orientation: new Cesium.VelocityOrientationProperty(posProp), model: { uri: ... } });
```
  - 说明：对每个节点按时间构建 `SampledPositionProperty`，配合 `VelocityOrientationProperty` 以自动计算朝向；实体使用不同模型（卫星/地面站/UAV）。

```js
viewer.clock.startTime = shared.startUtc.clone();
viewer.clock.stopTime = Cesium.JulianDate.addSeconds(shared.startUtc, maxTimeMs/1000, new Cesium.JulianDate());
```
  - 说明：根据数据最大时间设置时钟范围，确保回放在正确时间段内。

```js
viewer.zoomTo(viewer.entities);
```
  - 说明：初始缩放至所有实体以获得总览视角。

- **`create3DLinks(viewer)`（`src/vis/lib/links3d.js`）**
  - 概要：为每对实体创建一个 `Polyline`，并在 `scene.preRender` 中每帧更新位置与颜色（基于链路状态）。
  - 关键点：
    - 使用 `PolylineCollection` 可以在运行时高效更新大量线段属性；
    - 每帧遍历所有实体对，复杂度为 O(n^2)，在节点数量增多时会成为性能瓶颈；
    - 通过 `getLinkStatus(idA, idB, ms)` 获取状态以决定线条显示与颜色。

- **`update2DTopology(viewer)`（`src/vis/lib/topo2d.js`）**
  - 概要：使用 ECharts 的力导向图绘制 2D 拓扑，并根据当前时间动态设置边的状态显示。
  - 关键代码与说明：

```js
const ms = Cesium.JulianDate.secondsDifference(viewer.clock.currentTime, shared.startUtc) * 1000;
const edges = [];
shared.nodeInfoList.forEach((n1, i) => { shared.nodeInfoList.slice(i+1).forEach(n2 => { const status = getLinkStatus(n1.id, n2.id, ms); if (status) edges.push(...); }); });
```
  - 说明：以当前时钟时间计算 ms，再对所有节点对求状态并生成边集合；随后使用 `lastEdgesString` 避免重复渲染。

- **`getLinkStatus(idA, idB, currentTimeMs)`（`src/vis/lib/utils.js`）**
  - 概要：查找在 `currentTimeMs` 之前最近的链路事件并返回其 `status`。
  - 关键实现：

```js
const key = [String(idA), String(idB)].sort().join('---');
const lastEvent = shared.topologyEvents
    .filter(e => e.key === key && e.time <= currentTimeMs)
    .sort((a,b) => b.time - a.time)[0];
return lastEvent ? lastEvent.status : null;
```
  - 说明：对 `topologyEvents` 进行过滤与排序得到最近事件；若数据量大可考虑按 key 建索引以提高查询性能。

- **全局交互函数（`src/vis/main.js`）**
  - `toggleOption('ANA')`：切换分析面板显示，会调整 DOM 高度并在打开时调用 `update2DTopology(viewer)` 并触发 `shared.chart2D.resize()`。

```js
shared.state.showAnalytics = !shared.state.showAnalytics;
if (shared.state.showAnalytics) { anaView.style.height = '40%'; appView.style.height = '60%'; setTimeout(() => { shared.chart2D?.resize(); update2DTopology(viewer); }, 400); }
```
  - 说明：通过短延迟确保容器尺寸变更后 `echarts` 做正确的重绘。

  - `selectTarget(id)`：设置 `shared.state.currentTarget` 并通过 `viewer.trackedEntity` 跟随实体；若 `id` 为空则取消跟随并回到总览。

```js
shared.state.currentTarget = id || null;
if (!id) { viewer.trackedEntity = undefined; viewer.zoomTo(viewer.entities); return; }
const target = shared.entityMap.get(id);
if (target) { viewer.trackedEntity = target; viewer.trackedEntity.viewFrom = target.viewFrom || defaultViewFrom; }
```
