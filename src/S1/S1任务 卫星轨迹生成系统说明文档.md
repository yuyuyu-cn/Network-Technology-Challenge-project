S1任务是整个救援卫星通信仿真系统的基础数据层，核心输出为救援中心上空可见卫星的轨迹数据（CSV格式切片文件+索引文件），直接为S2任务的卫星通信链路仿真、资源调度等功能提供精准的卫星位置、状态等核心数据。

# 3. 系统核心功能

S1任务核心实现“卫星数据加载→动态筛选→轨迹计算→数据校验→文件输出”全流程，具体功能如下：

## 3.1 卫星数据加载

- 从CelesTrak官网下载的2026年3月5日 8:00 UTC的Starlink卫星TLE（两行轨道数据），并保存至本地文件（starlink.tle）。

## 3.2 卫星动态筛选

核心功能：基于救援中心（观察点）的坐标，按时间窗口动态筛选可见卫星，解决长时仿真中卫星“飞出/飞回”救援区域的问题。

- 筛选逻辑：当前时刻卫星仰角>0°（地平线以上）或卫星与救援中心距离<2000km，满足任一条件即视为可见。

- 动态机制：每60秒重新筛选一次可见卫星，每次筛选后保留距离救援中心最近的50颗卫星，确保轨迹数据始终对应当前可见卫星。

- 卫星元数据生成：为每颗筛选出的卫星分配唯一node_id（基于NORAD编号）、IP地址（基于NORAD编号生成，避免冲突），便于后续追踪和与S2任务对接。

## 3.3 卫星轨迹计算

基于动态筛选出的卫星列表，计算每个时间步（1秒/帧）的卫星轨迹数据，核心输出ECEF坐标（地心地固坐标系，单位：米）及卫星高度（单位：千米）。

- 时间范围：从仿真起始时刻（T0_UTC）开始，持续至仿真总时长结束，覆盖每一个时间步的卫星状态。

- 计算细节：利用skyfield库，结合DE421.bsp星历数据，将卫星的GCRS惯性系坐标转换为ITRS地固系（ECEF）坐标，同时计算卫星的星下点高度。

- 数据字段：包含时间戳（毫秒级）、卫星node_id、卫星名称、卫星类型、ECEF坐标（x/y/z）、卫星高度、轨道ID、IP地址、NORAD编号、卫星与救援中心距离等，满足项目规范及S2对接需求。

## 3.4 数据校验

对计算生成的轨迹数据进行多维度校验，确保数据符合项目规范，避免无效数据传入S2任务。

- 必填字段校验：检查轨迹数据是否包含所有必填字段（time_ms、node_id、name等10个核心字段），避免字段缺失。

- 时间戳连续性校验：确保时间戳覆盖所有时间步，无缺失、无重复，保证轨迹数据的时间完整性。

- 坐标合理性校验：检查卫星ECEF坐标对应的地心半径是否合理（不超过7000km），避免卫星坐标异常（如跑到外太空）。

- 空值校验：检查轨迹数据中是否存在空值，确保数据完整性。

## 3.5 文件输出

将校验通过的轨迹数据按时间切片保存为CSV文件，并生成索引文件（manifest.json），便于后续任务加载和使用。

- 切片规则：每60秒为一个切片，生成一个CSV文件，文件名格式为“sat_trace_{startMs}_{endMs}.csv”（startMs/endMs为切片时间范围的毫秒级时间戳）。

- 索引文件：包含仿真场景名称、仿真起始时间、仿真时长、卫星数量、所有CSV切片文件列表等信息，为S2任务提供数据索引。

- 输出路径：轨迹CSV文件保存至“StarCDN_Project/data/traces”目录，索引文件保存至“StarCDN_Project/data”目录（自动创建，无需手动操作）。

### 4.3.2 动态卫星筛选函数（filter_visible_satellites）

```python
def filter_visible_satellites(all_starlink_sats, observer, current_t):
    """
    单时间点筛选可见卫星
    筛选逻辑：当前时刻仰角>MIN_ALT_DEG 或 距离<MAX_DIST_KM
    """
    visible_sats = []
    for sat in all_starlink_sats:
        diff = sat - observer
        topo = diff.at(current_t)
        alt_deg = topo.altaz()[0].degrees
        dist_km = topo.distance().km

        if alt_deg > MIN_ALT_DEG or dist_km < MAX_DIST_KM:
            visible_sats.append((dist_km, sat))

    # 按距离排序取前N颗
    visible_sats_sorted = sorted(visible_sats, key=lambda x: x[0])
    selected_sats = visible_sats_sorted[:RESELECT_SAT_COUNT]
    
    # 生成动态元数据（保证node_id/IP相对稳定，优先复用已有ID）
    sat_metadata = []
    for idx, (dist_km, sat) in enumerate(selected_sats, 1):
        # 提取卫星唯一标识（NORAD编号），避免重复命名
        norad_id = sat.model.satnum
        sat_id = f"SAT_{norad_id:05d}"  # 用NORAD编号替代顺序号，保证唯一性
        ip = f"{IP_PREFIX}{norad_id % 255}"  # 基于NORAD编号生成IP，避免冲突
        
        sat_metadata.append({
            "node_id": sat_id,
            "name": sat.name.strip(),
            "ip": ip,
            "orbit_id": -1,
            "satellite_obj": sat,
            "norad_id": norad_id,  # 新增NORAD编号，便于追踪
            "current_dist_km": round(dist_km, 2)
        })
    return sat_metadata
```

功能：单时间点筛选可见卫星，生成卫星元数据，是动态筛选逻辑的核心。

流程：遍历所有Starlink卫星→计算卫星与救援中心的仰角和距离→筛选符合条件的卫星→按距离排序取前50颗→生成包含node_id、IP、NORAD编号等信息的元数据。

### 4.3.3 动态轨迹计算函数（calculate_dynamic_sat_trajectory）

```python
def calculate_dynamic_sat_trajectory(all_starlink_sats, ts, t0, observer):
    """
    动态计算卫星轨迹：每DYNAMIC_FILTER_INTERVAL_SEC秒重新筛选可见卫星
    """
    all_traces = []
    total_steps = SIM_DURATION_SEC // TIME_STEP_SEC

    # 预加载所有Starlink卫星（避免重复加载TLE）
    print(f"📡 预加载 {len(all_starlink_sats)} 颗Starlink卫星，开始动态轨迹计算...")

    for step in range(total_steps):
        current_sec = step * TIME_STEP_SEC
        current_time_ms = current_sec * MS_PER_SEC
        current_t = t0 + timedelta(seconds=current_sec)

        # 每N秒重新筛选一次可见卫星
        if current_sec % DYNAMIC_FILTER_INTERVAL_SEC == 0:
            current_sat_metadata = filter_visible_satellites(all_starlink_sats, observer, current_t)
            print(f"⏱️  时间 {current_sec}秒：筛选出 {len(current_sat_metadata)} 颗可见卫星")

        # 计算当前可见卫星的轨迹
        for sat_info in current_sat_metadata:
            sat = sat_info["satellite_obj"]
            geocentric = sat.at(current_t)
            ecef_xyz_m = geocentric.frame_xyz(itrs).m
            ecef_x, ecef_y, ecef_z = ecef_xyz_m

            subpoint = wgs84.subpoint(geocentric)
            altitude_km = subpoint.elevation.km

            trace = {
                "time_ms": current_time_ms,
                "node_id": sat_info["node_id"],
                "name": sat_info["name"],
                "type": "SAT",
                "ecef_x": round(ecef_x, 2),
                "ecef_y": round(ecef_y, 2),
                "ecef_z": round(ecef_z, 2),
                "altitude_km": round(altitude_km, 2),
                "orbit_id": sat_info["orbit_id"],
                "ip": sat_info["ip"],
                "norad_id": sat_info["norad_id"],  # 新增字段：卫星唯一标识
                "distance_km": sat_info["current_dist_km"]  # 新增字段：当前距离
            }
            all_traces.append(trace)

        # 进度提示（每小时输出一次）
        if current_sec % 3600 == 0 and current_sec > 0:
            print(f"🚀 已完成 {current_sec/3600} 小时轨迹计算，累计 {len(all_traces)} 条记录")

    print(f"📊 完成 {total_steps} 个时间步的轨迹计算，共 {len(all_traces)} 条记录")
    return pd.DataFrame(all_traces)
```

功能：基于动态筛选结果，计算每个时间步的卫星轨迹数据，是S1任务的核心业务函数。

关键逻辑：每60秒重新调用筛选函数更新卫星列表→遍历当前可见卫星→计算卫星ECEF坐标和高度→组装轨迹数据→累计所有时间步的轨迹数据，最终返回DataFrame。

### 4.3.4 数据校验函数（validate_trajectory_data）

```python
def validate_trajectory_data(df):
    """
    数据校验：确保符合项目规范
    """
    print("\n🔍 开始数据校验...")
    valid = True

    # 1. 检查必填字段
    required_cols = ["time_ms", "node_id", "name", "type", "ecef_x", "ecef_y", "ecef_z", "altitude_km", "orbit_id", "ip"]
    missing_cols = [col for col in required_cols if col not in df.columns]
    if missing_cols:
        print(f"❌ 缺少必填字段：{missing_cols}")
        valid = False

    # 2. 检查时间戳连续性
    time_steps = sorted(df["time_ms"].unique())
    expected_steps = list(range(0, SIM_DURATION_SEC * MS_PER_SEC, TIME_STEP_SEC * MS_PER_SEC))
    if time_steps != expected_steps:
        print(f"❌ 时间戳不连续！期望 {len(expected_steps)} 个步骤，实际 {len(time_steps)} 个")
        valid = False

    # 3. 检查ECEF坐标合理性（地球半径~6371km，卫星高度~550km，总半径~6921km）
    earth_radius_km = 6371
    max_expected_radius_km = 7000  # 最大允许半径（避免卫星跑到外太空）
    df["radius_km"] = np.sqrt(
        (df["ecef_x"]/1000)**2 + (df["ecef_y"]/1000)**2 + (df["ecef_z"]/1000)**2
    )
    abnormal_radius = df[df["radius_km"] > max_expected_radius_km]
    if not abnormal_radius.empty:
        print(f"❌ 发现 {len(abnormal_radius)} 条异常坐标（半径超过 {max_expected_radius_km}km）")
        valid = False

    # 4. 检查空值
    null_counts = df.isnull().sum()
    if null_counts.sum() > 0:
        print(f"❌ 存在空值：{null_counts[null_counts > 0].to_dict()}")
        valid = False

    if valid:
        print("✅ 数据校验通过！所有规范均满足")
    else:
        raise ValueError("数据不符合项目规范，请检查配置或代码")
```

功能：对轨迹数据进行多维度校验，确保数据符合项目规范，若校验失败则抛出异常，终止程序，避免无效数据输出。

### 4.3.5 文件输出函数（split_and_save_csv）

```python
def split_and_save_csv(trajectory_df):
    """
    按60秒切片保存CSV文件
    文件名格式：sat_trace_{startMs}_{endMs}.csv
    """
    # 创建输出目录（如果不存在）
    os.makedirs(OUTPUT_DIR, exist_ok=True)

    # 计算切片数量
    total_chunks = SIM_DURATION_SEC // CHUNK_DURATION_SEC

    for chunk_idx in range(total_chunks):
        # 切片时间范围（毫秒）
        start_sec = chunk_idx * CHUNK_DURATION_SEC
        end_sec = start_sec + CHUNK_DURATION_SEC
        start_ms = start_sec * MS_PER_SEC
        end_ms = end_sec * MS_PER_SEC - 1  # 闭区间：[startMs, endMs]

        # 筛选当前切片的数据
        chunk_df = trajectory_df[
            (trajectory_df["time_ms"] >= start_ms) &
            (trajectory_df["time_ms"] < end_ms + 1)
        ]

        # 文件名
        filename = f"sat_trace_{start_ms}_{end_ms}.csv"
        file_path = os.path.join(OUTPUT_DIR, filename)

        # 保存CSV（不保留索引）
        chunk_df.to_csv(file_path, index=False, encoding="utf-8")
        print(f"💾 保存切片文件：{filename}（{len(chunk_df)} 条记录）")

    # 生成manifest.json（总索引文件）
    manifest = {
        "scenario_name": "rescue_mission_2026_v1",
        "t0_utc": T0_UTC.strftime("%Y-%m-%d %H:%M:%S"),
        "sim_duration_sec": SIM_DURATION_SEC,
        "sat_count": MAX_SAT_COUNT,
        "trace_files": [
            f"sat_trace_{chunk_idx*CHUNK_DURATION_SEC*MS_PER_SEC}_"
            f"{(chunk_idx+1)*CHUNK_DURATION_SEC*MS_PER_SEC - 1}.csv"
            for chunk_idx in range(total_chunks)
        ]
    }
    manifest_path = os.path.join(
        os.path.dirname(OUTPUT_DIR), "manifest.json"
    )
    import json
    with open(manifest_path, "w", encoding="utf-8") as f:
        json.dump(manifest, f, indent=2)
    print(f"📋 生成索引文件：manifest.json")
```

功能：将轨迹数据按60秒切片保存为CSV文件，生成索引文件，确保输出格式符合S2任务的加载要求。

# 6. 注意事项

- 配置一致性：全局配置中需与S2协商的参数（T0_UTC、救援中心坐标等）必须严格一致，否则会导致S1与S2数据对接异常。

- TLE数据有效性：Starlink TLE数据每小时更新，长时仿真（如24小时）建议启用TLE自动下载功能，确保轨道数据精度；若使用本地旧文件，可能导致轨迹计算误差增大。

- 长时仿真优化：若将SIM_DURATION_SEC改为86400（24小时），建议将CHUNK_DURATION_SEC改为3600（1小时切片），避免生成过多小文件；同时可将TIME_STEP_SEC改为5秒，降低计算量和文件大小。

- 异常处理：程序运行中若出现“下载TLE失败”“数据校验失败”等异常，终端会输出具体错误信息，需根据提示排查（如检查网络、TLE文件、配置参数等）。

- DE421.bsp星历：skyfield会自动加载DE421.bsp星历（首次运行时自动下载），无需手动配置，该星历是保证轨迹计算精度的核心，不可删除。

# 7. 与S2任务对接说明

## 7.1 对接数据

S1任务向后续任务输出以下数据，供后续任务使用：

- 轨迹CSV切片文件：包含每个时间步的卫星位置、状态等核心数据，S3和S4可按时间切片加载，用于通信链路仿真。

- 索引文件（manifest.json）：后续任务可通过该文件获取仿真核心信息，快速定位所有轨迹CSV文件，实现数据批量加载。

## 7.2 对接注意事项

- 数据格式：S1输出的CSV文件字段、文件名格式、输出路径均严格遵循项目规范，后续人员可直接按对应格式加载，无需额外处理。

- 时间同步：S1与S2的仿真起始时间（T0_UTC）、时间步长（TIME_STEP_SEC）必须完全一致，确保卫星轨迹数据与S2的仿真时间线同步。

- 卫星标识：S1生成的卫星node_id（基于NORAD编号）、IP地址是S5识别卫星的唯一标识，双方需确保标识规则一致，避免通信链路匹配错误。

# 8. 常见问题排查

|常见问题|排查方法|
|---|---|
|程序启动失败，提示“本地无TLE文件且下载失败”|1. 检查网络是否通畅，启用TLE下载功能；2. 手动下载TLE文件，放入代码所在目录，命名为starlink.tle。|
|数据校验失败，提示“时间戳不连续”|检查SIM_DURATION_SEC和TIME_STEP_SEC配置，确保总时长能被时间步长整除；检查程序运行过程中是否被中断。|
|轨迹数据中卫星坐标异常（半径超过7000km）|检查TLE文件是否有效，可重新下载最新TLE数据；检查救援中心坐标配置是否正确。|
|长时仿真内存溢出|启用边计算边写入的轨迹计算方式（可基于现有代码修改）；增大时间步长；减少每次筛选的卫星数量。|
|与S2对接时，卫星标识不匹配|检查IP_PREFIX配置是否与S2一致；确认S1生成的node_id、IP地址规则与S2的识别规则一致。|
